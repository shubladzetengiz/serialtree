# SerialManager

[🇬🇪 ქართული](README.md) · [🇬🇧 English](README.en.md) · [🇷🇺 Русский](README.ru.md) · [🇫🇷 Français](README.fr.md)

სერიალებისა და ფილმების მართვის ვებ-აპლიკაცია. PHP + SQLite + Tailwind CSS.

![დემონსტრაცია](demo.png)

## მოთხოვნები

- PHP 8.1 ან უფრო ახალი
- ფუნქცია `mbstring` (PHP-ს ნაწილი)
- `php-sqlite3` პაკეტი

```
sudo apt install php-sqlite3 php-mbstring
```

## გაშვება

### ძირითადი (router-ით — რეკომენდებული)

```sh
php -S localhost:8002 -t serial2 serial2/router.php
```

### Router-ის გარეშე

```sh
php -S localhost:8002 -t serial2
```

Router გამოიყენება სტატიკური ფაილების (JS, CSS, სურათები) კორექტულად გასაცემად.

### Port-ის შეცვლა

`8002` შეგიძლიათ ნებისმიერი თავისუფალი პორტით ჩაანაცვლოთ.

## ფუნქციონალი

### ძირითადი
- სერიალებისა და ფილმების დამატება/რედაქტირება/წაშლა
- ორი რეჟიმი: ბადე (grid) და სია (list)
- ცოცხალი ძიება (Unicode-რეგისტრიგანურჩეველი, 300ms debounce)
- ფილტრი სტატუსის მიხედვით
- სორტირება: თარიღით (ახალი/ძველი) და ანბანით (ა–ჰ / ჰ–ა)

### Cover-ები (გარეკანები)
- გარე URL-დან გადმოწერა და ლოკალურად შენახვა
- ბეიჯი: ✓ Local / ✗ Online
- 100+ cover უკვე გადმოწერილია

### ავტო-სტატუსი
- შევსებულია ორივე ველი (სეზონი+ეპიზოდი) → `ნანახი`
- შევსებულია მხოლოდ ერთი → `გასაგრძელებელია`
- ორივე ცარიელია → `სანახავია`
- სეზონი > 99 → იწმინდება ორივე ველი, სტატუსი → `სანახავია`

### იმპორტი / ექსპორტი
- SQL ექსპორტი (ერთი ფაილი ორივე ტაბულისთვის)
- SQL იმპორტი (.sql, max 10MB, CSRF-დაცული)
- იმპორტი ფილტრავს მხოლოდ არსებულ კოლონკებს (უკუთავსებადია)

### ინტერფეისი
- 4 ენა: 🇬🇪 ქართული, 🇬🇧 English, 🇷🇺 Русский, 🇫🇷 Français
- მუქი თემა (Tailwind CSS + Font Awesome 6)
- ადაპტური: მაგიდა დესკტოპზე, ბარათები მობილურზე
- CSRF-დაცვა (POST-მოთხოვნებისთვის)
- წაშლა 5-წამიანი ტაიმერით

## მონაცემთა ბაზა

- ფაილი: `data.sqlite`
- ტაბულები: `series` (სერიალები) და `movies` (ფილმები)
- თვითმიგრაცია: გამოტოვებული კოლონკები ემატება ავტომატურად

### series

|---|---|---|
| id | INTEGER | მთავარი გასაღები |
| cover | TEXT | Cover-ის გზა/URL |

| episode | INTEGER | ეპიზოდი |
| status | TEXT | სტატუსი |
| rating | INTEGER | შეფასება (0–5) |
| resource_url | TEXT | წყაროს ბმული |

### movies

|---|---|---|

## მონაცემების აღდგენა

თუ `data.sqlite` წაიშალა (ან პირველად გაშვება), აპლიკაცია შემოგთავაზებთ:
1. **ცარიელი ბაზის შექმნას** — ავტომატური მიგრაცია
2. **SQL ფაილის იმპორტს** — წინასწარ გაკეთებული backup-იდან
3. **გაუქმებას**

SQL backup-ის შექმნა: ღილაკი `ექსპორტი` აპლიკაციის ზედა პანელში.

## უსაფრთხოება

- CSRF-ტოკენი ყველა POST მოთხოვნაზე
- SQL-ინექციისგან დაცვა: prepared statements
- XSS-დაცვა: `htmlspecialchars()` ყველა გამოტანაზე
- ფაილების იმპორტი: მხოლოდ `.sql`, max 10MB
